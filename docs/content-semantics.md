# 摸鱼内容语义

## 为什么不能只按作者和地区抽取

“摸鱼”不是名句展示器。用户选择发呆、喝茶、透气或散步时，真正发出的不是内容检索请求，而是一种很轻的心理请求：允许停下、获得安放、松开压力，或重新开始移动。

它也不是现实政治讨论场。会把用户重新拉回争议、立场和信息压力的近现代中国政治人物或事件，不进入这个情绪休息空间。

因此，同样写月亮的句子不能被视为同类。它可能在写独处的澄静、远方的思念、时间的无常，也可能是面对天地时的敬畏。地域、时代、语言负责说明文案从哪里来；语义标签负责说明它为什么应在此刻出现。

## 五层标签

| 层次 | 回答的问题 | 示例 |
| --- | --- | --- |
| 场景内核 | 意识被带入什么生活或心理场域？ | 山水天地、独处静室、行路迁徙、炉火日常、相逢共坐、故园回望、晨昏风雨、宇宙玄思、人间行役、梦与内景 |
| 情感内核 | 句子真正承载什么情绪张力？ | 澄静、温情、思念、怅惘、惊奇敬畏、欣悦、坚韧、旷达、归属、无常、释然 |
| 心理需要 | 用户读到它时，可能需要得到什么？ | 歇息、被接住、松绑、重新相连、拉远视角、重新开始、找回意义、继续向前、允许无用 |
| 文字动作 | 这句话用什么姿态靠近用户？ | 静静看见、轻声安抚、并肩陪伴、向外邀请、换个角度、唤醒感官、确认力量、打开空间 |
| 能量强度 | 它抵达用户时应该有多大力度？ | 静止、低柔、流动、明亮、稳定、舒展 |

每条文案有一至两个场景、一至两个情感内核，以及一个心理需要、文字动作和能量标签。标签保存在结构化语料和数据库维度关系中，不是只写在说明文档里的形容词。

## 语义推导

构建器先识别中、英、西三种语言中的场景与情感信号，再组合推导心理需要、文字动作和能量。比如：

- “嫦娥应悔偷灵药”不是因为出现天空就归为透气治愈，而是“晨昏风雨／梦与内景 → 怅惘 → 被接住 → 并肩陪伴 → 低柔”。
- “For love is sufficient unto love”是“相逢共坐／梦与内景 → 温情 → 重新相连 → 轻声安抚 → 低柔”。
- “Yo he visto en la noche oscura…”是“山水天地／晨昏风雨 → 惊奇敬畏 → 拉远视角 → 唤醒感官 → 明亮”。
- 没有明确情绪词的意象句，会由场景组合推断情感，但标记为 `contextual` 置信度，留在后续人工复核队列中，不伪装成确定结论。

当前实现是可复现、可检查的多语言词汇评分与组合规则。它适合现有公版诗歌语料；扩展到当代口语或万级内容时，应替换为“模型初标 + 人工复核”的离线管线，但维持相同的标签契约。

文案归属哪一种行为也由这五层语义决定。构建器会计算每条文案对四种行为的亲和度，再用“错配代价最大者优先”的容量分配保持四个行为数量均衡。质量检查要求至少 70% 落在语义最优行为、至少 80% 与最优值相差不超过 10 分，平均错配代价低于 5；文化配额不能靠明显的情绪错配来完成。

## 如何参与交互

运行时先按行为和文化来源生成每日平衡池，再在池内进行语义评分：

1. 发呆偏向梦与内景、独处静室、澄静、允许无用和静止能量。
2. 喝茶偏向炉火日常、相逢共坐、温情、归属、被接住和低柔能量。
3. 透气偏向山水天地、晨昏风雨、释然、松绑、打开空间和舒展能量。
4. 散步偏向行路迁徙、山水天地、旷达、坚韧、拉远视角和流动能量。
5. 清晨、白天、黄昏、深夜会分别提高明亮、稳定、怀旧和向内标签的权重。
6. 一分钟以内的短暂停留偏向歇息、松绑和低刺激；三分钟以上会提高视角、意义、更新与稳定力量的权重。
7. 最近出现过的内容继续降权，语义匹配不会牺牲新鲜度。

这套逻辑的目标不是猜中用户的心理诊断，而是在用户没有填写情绪问卷的前提下，让内容的姿态与他主动选择的休息方式保持一致。

## 当代论坛层

论坛内容不是把高赞评论搬进产品，而是把公开热门线程当作一份“今天人们在关心什么”的议题索引，再围绕其场景和情感内核写成原创中文微评论。这样既保留当代生活的温度，也避免用户名、私密经历、来源不明的段落和论坛攻击性进入休息空间。

- V2EX 偏向本土数字生活：AI 打断心流、应用隐私、职业倦怠、日常消费、兴趣班、备菜与数字怀旧。
- Hacker News 偏向技术与人文边界：职场监控、旧设备修复、太空科学、纸笔专注、开放技术与清晰设计。
- Reddit 只选友善生活社区：创作感恩、长假、友谊、园艺散步、自然衰老、无聊耐受与家庭传统。

每个论坛 40 条且各有轻松/深度 20 条；每种语气再按四种行为各分配 15 条。行为归属以人工语义剖面为先，不为了让单一论坛看起来整齐而制造明显情绪错配。轻松不是低俗段子，而是用克制幽默降低进入门槛；深度也不是说教，而是追问议题背后的注意力、关系、边界、代价和心理需要。

线程元数据记录在 `moyu-forum-copy.json`，正文权利状态统一为 `original-commentary`。当前版本观察于 2026-07-18，有效至 2026-10-18；前台只抽取未过期条目，运营台仍保留完整记录。热点到期后先更新脚本中的已复核线程清单、观察日期与版本，再运行 `npm run build:forums` 生成新冻结文件，并由同一质量检查复核。

## 质量门槛

`npm run check:quotes` 会验证：

- 1,000 条新增语料全部具备五层标签，且只使用受控词表。
- 标签无重复、无未知值，场景和情感在每个地区及每种行为中保持必要多样性。
- 心理需要、文字动作和能量不能被单一标签占据一半以上。
- 行为归属必须通过最优匹配率、近似最优率和平均错配代价检查。
- 中、英、西三语固定样本必须维持预期语义。
- `contextual` 条目被明确保留为人工复核队列，而不是提高伪置信度。
- 120 条论坛原创评论必须保持三个论坛、两种语气和四种行为的精确配额，并检查线程域名白名单、观察/过期日期、原创权利字段和非低俗词边界。
- 论坛正文与古典语料一起去重，并继续经过近现代中国政治人物及明显政治事件阻断检查。
