# 摸鱼设计系统 2026

## 产品判断

摸鱼不是效率工具，也不是冥想课程。它是一段由用户主动进入、随时可以离开的精神空间。产品的价值不在于教用户怎样休息，而在于承认这一小段时间本来就属于他。

因此界面遵守四条边界：不催促、不解释、不考核、不占有。没有连续任务、红点、进度、排名和劝导文案；一次只呈现一个动作、一句内容和一个明确出口。

## 交互骨架

1. 欢迎页只保留一次触摸仪式。鱼是进入空间的门，不是品牌吉祥物说明页。
2. 森林页只保留四个动作：发呆、喝茶、透气、散步。动作名称可见，说明文字隐藏在内容模型中。
3. 计时页只显示时长、声音、暂停和结束。计时是陪伴，不是倒计时。
4. 结果页只显示时间、原句、出处、城市换算和可能掉落的卡片。
5. 卡册与设置是浮在内容之上的控制层，不成为首页信息架构的主角。

## Apple 设计语言的取舍

参考 iOS 26 / Liquid Glass，但不复刻系统外观：

- 图片、森林和文字属于内容层，保持全屏、连续、可沉浸。
- 玻璃只用于动作组、底部工具、声音按钮和面板等控制层，不包裹大段内容。
- 关键标题使用粗体、左对齐和紧凑字距；文学内容使用系统宋体形成安静的阅读温度。
- 控件保持至少 44 点触摸面积，曲率与移动设备圆角形成同心关系。
- 动效表达空间连续性：鱼游入森林、页面淡入、面板从触点附近升起；系统开启“减少动态效果”时取消循环与位移动画。

## 四套素材哲学

### 发呆：虚与观

- 视觉：薄雾、湖面、远山、留白。
- 内容：庄子、老子、陶渊明、王维。
- 内在联系：不要求清空大脑，只允许意识暂时没有目的。

### 喝茶：温与安

- 视觉：窗、暖光、蒸汽、近距离生活痕迹。
- 内容：苏轼、白居易、陆游及茶诗。
- 内在联系：通过温度和手中之物，让抽象的休息落回身体。

### 透气：风与澄

- 视觉：林间光、流水、开阔天空。
- 内容：《庄子》、王维、杜甫、张孝祥。
- 内在联系：不要求情绪变好，只重新确认自己与外部世界仍有交换。

### 散步：行与归

- 视觉：路径、山谷、暮色、可继续向前的空间。
- 内容：苏轼、陶渊明、陆游、辛弃疾。
- 内在联系：走路不是运动目标，而是让脚步代替语言处理一会儿生活。

## 内容资产规则

- 当前库包含 4 条时段问候、880 条行为结果、320 条掉落卡片，共 1,204 条内容资产。
- 一千条跨文化语料按东亚、西方、南亚/中东、拉美分池，并同时记录时代、语言、作者、作品、公版状态与来源。
- 当代层新增 120 条论坛议题原创微评论：三个论坛各 40 条、轻松与深度各 60 条；论坛只提供议题线索，不复制用户名或评论正文。
- 每条结果与卡片还必须具备场景内核、情感内核、心理需要、文字动作和能量强度；地区说明它从哪里来，语义决定它为什么在此刻出现。
- “现代”优先使用已经进入公版的近现代原文；仍受保护的当代作者只进入待授权清单，不用来源不明的网摘补位。
- 情绪休息空间不承载现实政治表达；近现代中国党政军人物、政治活动人物及明显政治事件不进入作者、标题、正文或备注。
- 数据库保存完整内容库，前台每天按行为与类型轮换 144 条，近期出现过的内容继续降权；其中 32 条来自仍在有效期内的现代论坛层。
- 论坛层必须记录论坛、线程链接、议题、轻松/深度、观察日期与过期日期；到期后自动退出前台池，留在运营台等待复核或重建。
- 所有新增名句必须记录作者、篇名、原文语言、地区、时代、权利状态与原文核对来源。
- 单次只出现一句。海量属于后台能力，不属于前台密度。
- 详细标签契约与运行时匹配见 `content-semantics.md`。

## 素材资产规则

- 背景和图标必须是落入仓库的真实图片文件，不用 CSS 或组件路径临摹图形素材。
- CSS 只负责布局、材质、光线与响应状态。
- 图片不随内容高度拉伸，始终固定于视口并保持原始宽高比。
- 每个文件在 `moyu-asset-sources.md` 中保留来源、作者、许可和下载日期。
